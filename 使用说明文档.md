## 系统简介

本系统是一个基于 Web 的超市管理系统，用于管理超市的日常运营，包括商品管理、销售收银、进货入库、会员管理、供应商管理等功能。系统采用浏览器访问，无需安装客户端，打开网页即可使用。

### 系统特点
- **操作简单**：全中文界面，表格化展示，所见即所得
- **功能完整**：涵盖超市运营的核心流程
- **数据直观**：首页仪表盘一目了然查看经营状况
- **自动计算**：销售自动扣库存、进货自动加库存、会员自动积分

## 环境要求

在使用本系统前，电脑需要安装以下软件：

| 软件 | 最低版本 | 用途 |
|------|----------|------|
| Node.js | v16 以上 | 运行后端服务和前端开发服务器 |
| 浏览器 | 任意现代浏览器 | 访问系统页面（推荐 Edge 或 Chrome） |

### 如何检查是否已安装 Node.js

1. 按 `Win + R`，输入 `cmd`，回车
2. 在黑色窗口中输入：`node --version`
3. 如果显示类似 `v24.16.0` 的版本号，说明已安装
4. 如果提示"不是内部或外部命令"，需要先安装 Node.js

### 如何安装 Node.js

1. 打开浏览器，访问 https://nodejs.org
2. 下载左侧的 **LTS（长期支持版）** 安装包
3. 双击下载的 `.msi` 文件
4. 一路点击 **Next**，保持默认设置
5. 安装完成后重新打开命令窗口验证

## 安装与启动

### 方式一：使用桌面快捷脚本（推荐）

如果桌面上已有以下文件，直接使用即可：

1. **双击** 桌面上的 `启动超市管理系统.bat`
2. 等待弹出两个命令窗口（后端和前端）
3. 浏览器会自动打开系统页面
4. 使用完毕后，**双击** 桌面上的 `关闭超市管理系统.bat` 关闭服务

### 方式二：手动启动

1. 按 `Win + R`，输入 `cmd`，回车，打开命令窗口
2. 输入以下命令进入项目目录：
   ```
   cd C:\Users\王先宇\supermarket-system
   ```
3. 启动后端服务（保持窗口不要关闭）：
   ```
   node server/index.js
   ```
   看到 `超市管理系统后端已启动: http://localhost:3000` 表示成功
4. **再打开一个新的** 命令窗口，输入：
   ```
   cd C:\Users\王先宇\supermarket-system
   node .\node_modules\vite\bin\vite.js
   ```
   看到 `Local: http://localhost:5173/` 表示成功
5. 打开浏览器，访问 `http://localhost:5173`

### 关闭系统

- 方式一：双击桌面的 `关闭超市管理系统.bat`
- 方式二：分别关闭后端和前端的命令窗口（黑色窗口）

## 功能使用教程

### 1. 首页仪表盘

打开系统后默认显示首页仪表盘，包含以下信息：

#### 统计卡片（页面顶部）
- **今日销售额**：当天所有销售单的总金额
- **今日订单**：当天的销售笔数
- **商品总数**：系统中所有商品的数量
- **会员总数**：注册会员的数量

#### 库存预警（左下方）
- 显示库存低于预警值的商品列表
- 库存为 0 的商品显示红色标签
- 库存低于预警值的显示黄色标签
- 点击左侧菜单可直接跳转到商品管理进行补货

#### 热销排行（右下方）
- 显示销量最高的前 5 名商品
- 帮助了解哪些商品最受欢迎

---

### 2. 分类管理

用于管理商品的分类（如：食品饮料、日用百货等）。

#### 查看分类
1. 点击左侧菜单 **"分类管理"**
2. 页面显示所有分类的表格，包含 ID、名称、创建时间

#### 新增分类
1. 点击右上角 **"新增分类"** 按钮
2. 在弹出的对话框中输入分类名称
3. 点击 **"保存"** 按钮

#### 编辑分类
1. 在表格中找到要修改的分类
2. 点击该行右侧的 **"编辑"** 按钮
3. 修改名称后点击 **"保存"**

#### 删除分类
1. 点击该行右侧的 **"删除"** 按钮
2. 在确认对话框中点击 **"确定"**
3. **注意**：如果该分类下有商品，无法删除，需要先将商品移到其他分类或删除商品

---

### 3. 商品管理

用于管理超市的所有商品信息。

#### 查看与搜索商品
1. 点击左侧菜单 **"商品管理"**
2. 页面显示所有商品的表格
3. **搜索**：在右上角搜索框输入商品名称或条码，按回车或点击搜索
4. **分类筛选**：点击分类下拉框，选择某个分类只显示该分类的商品

#### 新增商品
1. 点击右上角 **"新增商品"** 按钮
2. 在弹出的对话框中填写：
   - **商品名称**（必填）：如"可口可乐 330ml"
   - **条码**（选填）：商品的条形码编号
   - **分类**（选填）：从下拉列表选择
   - **售价**（选填）：卖给顾客的价格
   - **成本价**（选填）：进货的价格
   - **库存**（选填）：当前库存数量
   - **预警值**（选填）：库存低于此值时在仪表盘显示预警，默认 10
   - **单位**（选填）：个/瓶/袋/盒等
3. 点击 **"保存"**

#### 编辑商品
1. 找到要修改的商品行
2. 点击 **"编辑"** 按钮
3. 修改信息后点击 **"保存"**

#### 删除商品
1. 点击 **"删除"** 按钮
2. 确认后删除

#### 库存说明
- 库存数量会通过 **进货管理** 自动增加
- 库存数量会通过 **销售收银** 自动减少
- 也可以在编辑商品时手动修改库存

---

### 4. 销售收银

这是系统的核心功能，用于日常收银结算。

#### 操作流程

**第一步：搜索商品**
1. 点击左侧菜单 **"销售收银"**
2. 在搜索框中输入商品名称或条码
3. 按回车或点击 **"搜索"** 按钮
4. 搜索结果会显示在搜索框下方

**第二步：添加到购物车**
1. 在搜索结果中点击要购买的商品
2. 商品会自动添加到左侧的购物车表格中
3. 同一商品多次点击会累加数量
4. 可以用购物车中的 **数量按钮** 调整购买数量
5. 可以点击 **红色删除按钮** 移除购物车中的商品

**第三步：选择会员（可选）**
1. 在右侧 **"会员"** 下拉框中选择会员
2. 选择会员后，结算金额会自动转为积分（1元=1积分）
3. 如果不是会员，跳过此步骤

**第四步：选择支付方式**
- 💵 现金
- 📱 微信
- 📱 支付宝

**第五步：结算**
1. 确认右侧显示的商品数量和合计金额无误
2. 点击 **"确认结算"** 按钮
3. 在弹出的确认框中点击 **"确定"**
4. 系统自动完成：
   - 扣减商品库存
   - 记录销售单
   - 如果有会员，自动增加积分

#### 注意事项
- 如果商品库存不足，系统会提示并阻止结算
- 结算完成后购物车自动清空
- 可在页面右下方查看最近的销售记录

---

### 5. 进货管理

用于记录从供应商处进货的信息。

#### 查看进货记录
1. 点击左侧菜单 **"进货管理"**
2. 页面显示所有进货单，包含单号、供应商、金额、时间
3. 点击 **"查看"** 可以看到进货单的详细商品列表

#### 新建进货单
1. 点击右上角 **"新建进货单"** 按钮
2. **选择供应商**：从下拉列表中选择（可不选）
3. **添加商品**：
   - 点击 **"+ 添加商品"** 按钮
   - 在商品下拉框中搜索选择要进货的商品
   - 填写 **数量**（进了多少）
   - 填写 **单价**（每个的进货成本）
   - 可以多次点击添加多个商品
4. 页面下方会自动计算 **合计金额**
5. 点击 **"确认入库"** 按钮
6. 系统自动完成：
   - 记录进货单
   - 增加商品库存数量
   - 更新商品的成本价

#### 注意事项
- 进货后库存会自动增加，无需手动修改商品
- 进货单价会自动更新商品的成本价记录

---

### 6. 会员管理

用于管理超市的会员信息。

#### 查看与搜索会员
1. 点击左侧菜单 **"会员管理"**
2. 页面显示所有会员，包含姓名、手机号、积分、累计消费
3. 可以按姓名或手机号搜索

#### 新增会员
1. 点击 **"新增会员"** 按钮
2. 填写 **姓名**（必填）和 **手机号**（必填）
3. 点击 **"保存"**

#### 编辑会员
1. 找到要修改的会员
2. 点击 **"编辑"** 修改姓名或手机号

#### 删除会员
1. 点击 **"删除"** 按钮并确认

#### 积分规则
- 会员消费时自动积分，1元 = 1积分
- 积分和累计消费在销售收银时自动更新，无需手动操作

用于管理进货的供应商信息。

- 找到对应供应商，点击 **"编辑"** 或 **"删除"** 按钮

## 常见问题与解决

### Q1: 打开浏览器显示"无法访问此网站"
**原因**：后端或前端服务没有启动
**解决**：
1. 检查是否有两个命令窗口在运行
2. 如果没有，重新运行启动脚本
3. 如果有报错信息，检查端口是否被占用

### Q2: 页面上看不到数据，一直加载中
**原因**：后端服务未启动或启动失败
**解决**：
1. 查看后端命令窗口是否有红色报错
2. 如果报错 `EADDRINUSE`，说明端口 3000 被占用
3. 关闭占用端口的程序，或重启电脑后重试

### Q3: 搜索商品没有结果
**原因**：搜索关键词不匹配
**解决**：
1. 尝试只输入商品名称的一部分（如搜"可乐"而不是"可口可乐330ml"）
2. 检查商品是否已通过商品管理添加到系统中

### Q4: 销售时提示"库存不足"
**原因**：商品库存数量不够
**解决**：
1. 通过 **进货管理** 为该商品补货
2. 或在 **商品管理** 中手动编辑该商品的库存数量

### Q5: 删除分类时提示"该分类下有商品"
**原因**：该分类下还有商品，不能直接删除
**解决**：
1. 先到 **商品管理** 把该分类下的商品移到其他分类
2. 或删除该分类下的商品后再删除分类

### Q6: 如何备份数据？
**说明**：所有数据存储在 `C:\Users\王先宇\supermarket-system\server\supermarket.db` 这个文件中
**备份方法**：复制该文件到安全位置即可
**恢复方法**：将备份文件复制回原位置，重启服务

### Q7: 如何重新开始（清空所有数据）？
1. 关闭后端服务
2. 删除 `C:\Users\王先宇\supermarket-system\server\supermarket.db` 文件
3. 重新启动后端服务，系统会自动创建新的数据库并插入示例数据
