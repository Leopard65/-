# 流浪动物救助与领养管理系统 - 项目架构说明

本文档按当前代码结构整理，用于后续开发、答辩说明和交付到目标电脑运行。

## 1. 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | Vue 3、Vite、Element Plus、Vue Router、Pinia、Axios |
| 后端 | Node.js、Express、MySQL2、JWT、bcryptjs、multer |
| 数据库 | MySQL 8.0、utf8mb4 字符集 |

## 3. 前端架构

### 路由层

路由配置位于 `client/src/router/index.js`。

| 类型 | 路径 | 说明 |
|------|------|------|
| 前台 | `/` | 首页 |
| 前台 | `/animals`、`/animals/:id` | 动物列表与详情 |
| 前台 | `/adopt/apply/:id`、`/adopt/mine` | 领养申请与我的申请，需要登录 |
| 前台 | `/rescue` | 救助求助提交 |
| 前台 | `/articles`、`/articles/:id` | 科普文章 |
| 用户 | `/login`、`/register` | 登录注册 |
| 后台 | `/admin/**` | 后台管理，需要登录且角色为 `admin` |

路由守卫直接读取 `localStorage` 中的 `token` 和 `user`：

- 未登录访问需要登录的页面，会跳转到 `/login`。
- 非管理员访问 `/admin`，会回到首页。

### 请求层

`client/src/utils/request.js` 负责：

- Axios `baseURL` 为 `/api`；
- 自动在请求头携带 `Authorization: Bearer <token>`；
- 统一处理后端 `{ code, message, data }` 响应；
- 遇到 401 时清理登录信息并跳转登录页。

### 布局层

- `FrontLayout.vue`：前台页面统一导航和内容容器。
- `AdminLayout.vue`：后台侧边栏、顶部区域和管理页面出口。

## 4. 后端架构

后端入口为 `server/app.js`：

1. 加载 CORS、JSON、表单解析中间件；
2. 通过 `/uploads` 暴露上传图片；
3. 挂载 `/api/auth`、`/api/animals`、`/api/adoptions` 等业务路由；
4. 提供 `/api/health` 健康检查；
5. 使用统一错误处理中间件。

### 主要路由

| 路由前缀 | 文件 | 说明 |
|----------|------|------|
| `/api/auth` | `routes/auth.js` | 注册、登录、个人信息、改密 |
| `/api/animals` | `routes/animals.js` | 动物列表、详情、统计、后台 CRUD |
| `/api/adoptions` | `routes/adoptions.js` | 领养申请、审核、我的申请、回访记录 |
| `/api/rescues` | `routes/rescues.js` | 救助提交、后台处理、详情 |
| `/api/content` | `routes/content.js` | 公告、文章、轮播图 |
| `/api/users` | `routes/users.js` | 后台用户管理 |
| `/api/categories` | `routes/categories.js` | 动物分类与品种管理 |

### 权限模型

- `authenticate`：要求登录并校验 JWT。
- `requireAdmin`：要求当前用户角色为 `admin`。
- `optionalAuth`：可登录也可匿名，用于救助求助提交。

### 上传文件

上传由 `server/middlewares/upload.js` 处理：

- 支持 `jpg/png/gif/webp`；
- 单文件最大 5MB；
- 按日期写入 `server/uploads/YYYY-MM-DD/`；
- Web 访问路径由 `server/utils/file.js` 生成，格式为 `/uploads/YYYY-MM-DD/filename.ext`。

## 5. 数据库结构

初始化脚本：`database/init.sql`。推荐通过 `server/scripts/init-db.js` 执行，避免导入编码和管理员密码问题。

| 表 | 说明 |
|----|------|
| `users` | 用户、管理员、登录状态 |
| `animal_categories` | 动物分类，如猫、狗、兔 |
| `animal_breeds` | 动物品种 |
| `animals` | 动物主档案 |
| `adoption_applications` | 领养申请 |
| `adoption_followups` | 领养回访记录 |
| `rescue_requests` | 救助求助 |
| `announcements` | 公告 |
| `articles` | 科普文章、领养指南、故事 |
| `banners` | 首页轮播图 |

默认管理员：`admin / admin123`。脚本重复执行会把 admin 密码重置为 `admin123`，方便目标电脑找回后台入口。

## 6. 开发与交付边界

当前项目的开发电脑和运行电脑可以不同：

- 当前电脑负责继续开发、构建检查、整理文档和打包。
- 目标电脑负责重新安装依赖、配置 `.env`、初始化数据库并运行。

交付时不要携带：

- `node_modules/`
- `client/dist/`
- `server/uploads/`
- `server/.env`

目标电脑应从 `.env.example` 复制生成 `.env`，并填写自己的 MySQL 密码和 JWT 密钥。

## 7. 后续扩展建议

优先级较高的方向：

1. 后台仪表盘继续增强，例如趋势图、待处理事项列表、最近领养申请。
2. 领养回访增加下次回访日期、提醒状态和图片上传。
3. 救助模块增加地图定位或经纬度字段。
4. 前端抽取 API 模块和通用表格/表单组件，降低后台页面重复代码。
5. 增加基础测试或接口自检脚本，提升交付稳定性。
