# 部署与交付说明（目标电脑运行指南）

本文档面向**接收方电脑**：该电脑已安装 Node.js、MySQL、Git、VS Code，需要把本项目跑起来。
按顺序照做即可。文中 `<项目根目录>` 指本项目复制到目标电脑后的实际路径（如 `D:\projects\stray-animal-rescue`）。

---

## 1. 环境要求

| 软件 | 版本 | 验证命令 |
|------|------|----------|
| Node.js | 18 / 20 LTS 及以上 | `node --version` |
| npm | 9+ | `npm --version` |
| MySQL | 8.0 | `mysql --version` |

> 建议先配置 npm 国内镜像加速安装：`npm config set registry https://registry.npmmirror.com`

---

## 2. 安装依赖

```bash
cd <项目根目录>\server
npm install

cd <项目根目录>\client
npm install
```

---

## 3. 配置环境变量（后端）

```bash
cd <项目根目录>\server
copy .env.example .env
```

编辑 `server\.env`：

- `DB_PASSWORD`：改成本机 MySQL 的 root 密码。
- `JWT_SECRET`：改成一段随机字符串（`.env.example` 内含生成命令）。
- 其余保持默认即可（端口 3000、数据库名 `stray_animal_rescue`）。

> `.env` 已被 `.gitignore` 忽略，不会进入版本库，请勿把真实密码提交到 Git。

---

## 4. 初始化数据库

二选一，**推荐方式 A**：

**方式 A：脚本初始化（推荐，自动建库建表并写入管理员）**
```bash
cd <项目根目录>\server
npm run init-db
```
看到 “🎉 数据库初始化完成！管理员账号: admin / admin123” 即成功。脚本可重复执行，不会报错或产生重复数据。

**方式 B：直接导入 SQL**
```bash
mysql -u root -p --default-character-set=utf8mb4 < <项目根目录>\database\init.sql
```
> 必须带 `--default-character-set=utf8mb4`，否则中文会乱码。

验证：
```sql
USE stray_animal_rescue;
SHOW TABLES;          -- 应看到 10 张表
SELECT username, role FROM users;   -- 应有 admin / admin
```

---

## 5. 启动

**后端**（窗口 1）：
```bash
cd <项目根目录>\server
npm run dev      # 开发模式（nodemon 自动重启）；或 npm start
```
出现 “✅ MySQL 数据库连接成功” 表示数据库连通。后端：http://localhost:3000 （健康检查 `/api/health`）

**前端**（窗口 2）：
```bash
cd <项目根目录>\client
npm run dev
```
前端：http://localhost:5173

> 前端通过 `vite.config.js` 的代理把 `/api`、`/uploads` 转发到后端 3000 端口，因此开发模式下**必须先启动后端**。

## 6. 默认账号

| 角色 | 入口 | 账号 | 密码 |
|------|------|------|------|
| 管理员 | http://localhost:5173/admin | `admin` | `admin123` |
| 普通用户 | http://localhost:5173/register 自行注册 | — | — |

> 首次登录后建议在数据库或后续功能中修改管理员密码。重新执行 `npm run init-db` 会把 admin 密码重置回 `admin123`。

## 7. 前端生产构建（可选，用于正式部署）

```bash
cd <项目根目录>\client
npm run build       # 产物输出到 client/dist
```
`dist` 是纯静态文件，需由 Nginx / IIS / `serve` 等托管，并将 `/api`、`/uploads` 反向代理到后端 3000 端口（开发期由 Vite 代理完成，生产期需自行配置）。仅做答辩演示时，直接用 `npm run dev` 跑前端即可，无需构建。

---

## 8. 打包交付（从开发电脑发往目标电脑）

**需要带走：**
- `client/`（不含 `node_modules`、`dist`）
- `server/`（不含 `node_modules`、`uploads`、`.env`）
- `database/`、`docs/`、`README.md`、`.gitignore`

**不要带走 / 不要提交：**
| 目录/文件 | 原因 | 处理 |
|-----------|------|------|
| `node_modules/` | 体积大、平台相关 | 目标电脑 `npm install` 重新生成 |
| `client/dist/` | 构建产物 | 目标电脑按需 `npm run build` |
| `server/uploads/` | 用户上传的运行期数据 | 目标电脑首次运行自动创建 |
| `server/.env` | 含数据库密码、JWT 密钥 | 目标电脑用 `.env.example` 复制后自行填写 |

> 已提供根目录 `.gitignore`，用 Git 管理时上述内容会自动被忽略。若用压缩包交付，打包前请手动删除 `node_modules`、`dist`、`uploads`、`.env`。

## 9. 常见问题排查

| 现象 | 原因与解决 |
|------|------------|
| 后端报 `❌ MySQL 数据库连接失败` | `.env` 的 `DB_PASSWORD` 不对，或 MySQL 服务未启动（服务名通常为 `MySQL80`）。 |
| 后台 `admin/admin123` 登录失败 | 数据库未初始化或用了旧脚本；重新执行 `npm run init-db`。 |
| 页面中文或数据库中文乱码 | 用方式 B 导入时漏了 `--default-character-set=utf8mb4`；改用方式 A 或重新导入。 |
| 前端页面空白 / 接口 404 | 后端没启动，或没走 Vite 代理；确认后端在 3000 端口运行。 |
| 端口被占用 | 改 `server\.env` 的 `PORT`，或 `client\vite.config.js` 的 `server.port`（改端口后注意同步代理 target）。 |
| 上传的图片不显示 | 确认后端在运行（图片经 `/uploads` 由后端提供）；`server/uploads` 会自动创建。 |
| `npm install` 很慢 | `npm config set registry https://registry.npmmirror.com` 后重试。 |
