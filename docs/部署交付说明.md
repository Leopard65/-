# 部署与交付说明（当前电脑开发，目标电脑运行）

本文档面向两件事：

1. 在当前电脑继续开发、整理、打包项目。
2. 把项目交给另一台已经安装 Node.js、MySQL、Git、VS Code 的目标电脑运行。

文中 `<项目根目录>` 指本项目复制到目标电脑后的实际路径（如 `D:\projects\stray-animal-rescue`）。

---

## 1. 环境要求

| 软件 | 版本 | 验证命令 |
|------|------|----------|
| Node.js | 18 / 20 LTS 及以上 | `node --version` |
| npm | 9+ | `npm --version` |
| MySQL | 8.0 | `mysql --version` |

> 建议先配置 npm 国内镜像加速安装：`npm config set registry https://registry.npmmirror.com`

---

## 2. 安装依赖

如果目标电脑之前已经运行过旧版本，也建议重新执行一次 `npm install`，让依赖与 `package-lock.json` 保持一致。

```bash
cd <项目根目录>\server
npm install

cd <项目根目录>\client
npm install
```

---

## 3. 配置环境变量（后端）

```bash
cd <项目根目录>\server
copy .env.example .env
```

编辑 `server\.env`：

- `DB_PASSWORD`：改成本机 MySQL 的 root 密码。
- `JWT_SECRET`：改成一段随机字符串（`.env.example` 内含生成命令）。
- 其余保持默认即可（端口 3000、数据库名 `stray_animal_rescue`）。

> `.env` 已被 `.gitignore` 忽略，不会进入版本库，请勿把真实密码提交到 Git。

---

## 4. 初始化数据库

二选一，**推荐方式 A**：

**方式 A：脚本初始化（推荐，自动建库建表并写入管理员）**
```bash
cd <项目根目录>\server
npm run init-db
```
看到 “🎉 数据库初始化完成！管理员账号: admin / admin123” 即成功。脚本可重复执行，不会报错或产生重复数据。

**方式 B：直接导入 SQL**
```bash
mysql -u root -p --default-character-set=utf8mb4 < <项目根目录>\database\init.sql
```
> 必须带 `--default-character-set=utf8mb4`，否则中文会乱码。

验证：
```sql
USE stray_animal_rescue;
SHOW TABLES;          -- 应看到 13 张表
SELECT username, role FROM users;   -- 应有 admin / admin
```

> **旧库升级（重要）**：若目标电脑上已存在旧版本数据库，**直接重新执行 `npm run init-db` 即可完成升级**——脚本幂等，会自动补建新增表（`animal_events` 动物档案事件、`rescue_logs` 救助处理日志），并为旧表补充新列（`adoption_followups.next_visit_date` 下次计划回访日期），不会影响既有业务数据（仅会把 admin 密码重置为 `admin123`）。

### 4.1 可选：灌入演示数据（用于答辩/演示）

初始化后数据库只有管理员和基础分类，前台/仪表盘内容为空。如需一份现成的演示数据
（动物及档案事件 / 相册、文章、轮播、公告、救助及处理日志、领养 + 回访 + 计划回访 + 完成，
含仪表盘三类图表与回访提醒效果），执行：

```bash
cd <项目根目录>\server
npm run seed
```

会额外创建演示用户 `demo / demo123`。脚本**仅在动物表为空时执行**，避免重复或覆盖真实数据；
演示图片为 `client/public/seed/` 下的占位图，随前端同源提供，无需联网。

## 5. 启动

**后端**（窗口 1）：
```bash
cd <项目根目录>\server
npm run dev      # 开发模式（nodemon 自动重启）；或 npm start
```
出现 “✅ MySQL 数据库连接成功” 表示数据库连通。后端：http://localhost:3000 （健康检查 `/api/health`）

**前端**（窗口 2）：
```bash
cd <项目根目录>\client
npm run dev
```
前端：http://localhost:5173

> 前端通过 `vite.config.js` 的代理把 `/api`、`/uploads` 转发到后端 3000 端口，因此开发模式下**必须先启动后端**。

## 6. 默认账号

| 角色 | 入口 | 账号 | 密码 |
|------|------|------|------|
| 管理员 | http://localhost:5173/admin | `admin` | `admin123` |
| 普通用户 | http://localhost:5173/register 自行注册 | — | — |

> 首次登录后建议在数据库或后续功能中修改管理员密码。重新执行 `npm run init-db` 会把 admin 密码重置回 `admin123`。

## 7. 前端生产构建与单端口部署（推荐用于正式部署）

```bash
cd <项目根目录>\client
npm run build       # 产物输出到 client/dist
```

构建完成后，**后端会自动托管前端**：只要 `client/dist` 存在，启动后端即可在
`http://localhost:3000` 直接访问完整网站，`/api`、`/uploads` 与页面天然同源，
**无需 Vite 代理、无需 Nginx、无需开两个终端**。

```bash
cd <项目根目录>\server
npm start            # 浏览器打开 http://localhost:3000 即是完整前台/后台
```

> - 后端对所有非 `/api`、`/uploads` 的 GET 请求回退到 `index.html`，支持前端 history 路由（刷新子页面不会 404）。
> - 若 `client/dist` 不存在（未构建），访问 `http://localhost:3000/` 会显示一段友好提示，引导你用开发模式或先构建，而不是 `Cannot GET /`。
> - 仅做答辩演示且想要热更新时，仍可用「后端 `npm run dev` + 前端 `npm run dev`」的双服务开发模式（见第 5 节），浏览器开 5173。
>
> 如确需用独立的 Nginx / IIS / `serve` 托管 `dist`，记得把 `/api`、`/uploads` 反向代理到后端 3000 端口。

## 8. 打包交付（从开发电脑发往目标电脑）

### 8.1 一键生成交付包（推荐）

项目根目录已提供 PowerShell 打包脚本，会自动排除 `node_modules`、`.env`、上传目录、构建临时目录、
测试截图/报告和 PPT 构建素材，避免把本机隐私配置或大体积临时文件发到目标电脑。

先预览将要打包的内容：
```powershell
cd <项目根目录>
powershell -ExecutionPolicy Bypass -File .\scripts\package-delivery.ps1 -DryRun
```

生成源码交付包（目标电脑重新安装依赖并运行）：
```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\package-delivery.ps1
```

生成包含前端生产构建的交付包（目标电脑可直接用后端单端口访问 3000）：
```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\package-delivery.ps1 -BuildFrontend
```

生成后的 zip 位于：
```text
<项目根目录>\release\stray-animal-rescue_年月日_时分.zip
```

> `-BuildFrontend` 会先执行 `client\npm run build`，并把 `client/dist` 放入交付包。目标电脑仍需在
> `server` 执行 `npm install`、配置 `.env`、初始化数据库，然后 `npm start`。

### 8.2 手动打包清单

**需要带走：**
- `client/`（保留 `package.json`、`package-lock.json`，不含 `node_modules`、`dist`）
- `server/`（保留 `package.json`、`package-lock.json`、`.env.example`，不含 `node_modules`、`uploads`、`.env`）
- `database/`、`docs/`、`README.md`、`.gitignore`
- `scripts/`（包含交付打包脚本）
- `.git/` 可选：如果要在目标电脑继续用 Git 追踪历史，可以带；只做运行演示则不必带。

**不要带走 / 不要提交：**
| 目录/文件 | 原因 | 处理 |
|-----------|------|------|
| `node_modules/` | 体积大、平台相关 | 目标电脑 `npm install` 重新生成 |
| `client/dist/` | 构建产物 | 目标电脑按需 `npm run build` |
| `server/uploads/` | 用户上传的运行期数据 | 目标电脑首次运行自动创建 |
| `server/.env` | 含数据库密码、JWT 密钥 | 目标电脑用 `.env.example` 复制后自行填写 |

> 已提供根目录 `.gitignore`，用 Git 管理时上述内容会自动被忽略。若用压缩包交付，打包前请手动删除 `node_modules`、`dist`、`uploads`、`.env`。

目标电脑收到项目后，应重新执行：

```bash
cd <项目根目录>\server
npm install
copy .env.example .env
npm run init-db

cd <项目根目录>\client
npm install
npm run dev
```

## 9. 常见问题排查

| 现象 | 原因与解决 |
|------|------------|
| 后端报 `❌ MySQL 数据库连接失败` | `.env` 的 `DB_PASSWORD` 不对，或 MySQL 服务未启动（服务名通常为 `MySQL80`）。 |
| 后台 `admin/admin123` 登录失败 | 数据库未初始化或用了旧脚本；重新执行 `npm run init-db`。 |
| 页面中文或数据库中文乱码 | 用方式 B 导入时漏了 `--default-character-set=utf8mb4`；改用方式 A 或重新导入。 |
| 前端页面空白 / 接口 404 | 后端没启动，或没走 Vite 代理；确认后端在 3000 端口运行。 |
| 端口被占用 | 改 `server\.env` 的 `PORT`，或 `client\vite.config.js` 的 `server.port`（改端口后注意同步代理 target）。 |
| 上传的图片不显示 | 确认后端在运行（图片经 `/uploads` 由后端提供）；`server/uploads` 会自动创建。 |
| `npm install` 很慢 | `npm config set registry https://registry.npmmirror.com` 后重试。 |
