# 快速启动指南（目标电脑运行）

本文档用于在**另一台目标电脑**上快速跑起项目。目标电脑如果已经安装好 Node.js、MySQL、Git、VS Code，可以直接从“第二步”开始。

## 第一步：安装环境
按照 `docs/环境安装教程.md` 依次安装以下软件：
1. ✅ Node.js（LTS 版本）
2. ✅ Git
3. ✅ MySQL 8.0
4. ✅ VS Code
5. ✅ Chrome

安装完后打开 cmd 验证：
```bash
node --version
npm --version
git --version
mysql --version
```

## 第二步：配置 npm 镜像
```bash
npm config set registry https://registry.npmmirror.com
```

> 说明：下文用 `<项目根目录>` 表示你把本项目复制到目标电脑后的实际路径，
> 例如 `D:\projects\stray-animal-rescue`。请把命令里的 `<项目根目录>` 换成真实路径，
> 或先 `cd` 进入该目录再执行后续命令。
>
> 从开发电脑拷贝项目时，不需要带 `node_modules`、`client/dist`、`server/uploads` 和 `server/.env`。
> 目标电脑会重新安装依赖，并从 `.env.example` 复制自己的 `.env`。

## 第三步：配置后端并初始化数据库
1. 打开 cmd，进入后端目录并安装依赖：
   ```bash
   cd <项目根目录>\server
   npm install
   ```
2. 复制配置文件：
   ```bash
   copy .env.example .env
   ```
3. **编辑 `server\.env`**：
   - 把 `DB_PASSWORD` 改为你安装 MySQL 时设置的 root 密码；
   - 把 `JWT_SECRET` 改为一段随机字符串（文件内有生成命令）。
4. 初始化数据库（**推荐**，会自动建库、建表、写入管理员和初始数据，且可重复执行）：
   ```bash
   npm run init-db
   ```
   看到 “🎉 数据库初始化完成！管理员账号: admin / admin123” 即成功。

   > 备选方案（不想用脚本时）：登录 MySQL 后直接导入 SQL —
   > `mysql -u root -p --default-character-set=utf8mb4 < <项目根目录>\database\init.sql`
   > 务必带上 `--default-character-set=utf8mb4`，否则中文可能乱码。两种方式选其一即可。

## 第四步：启动后端
在后端目录执行：
```bash
npm run dev          # 开发模式，自动重启
# 或 npm start       # 生产模式
```
看到 “✅ MySQL 数据库连接成功” 和 “🐾 后端已启动” 就成功了。后端地址：http://localhost:3000

## 第五步：启动前端
1. 打开另一个 cmd 窗口：
   ```bash
   cd <项目根目录>\client
   npm install
   npm run dev
   ```
2. 浏览器打开 http://localhost:5173

## 第六步：测试
- 前台首页：http://localhost:5173
- 后台管理：http://localhost:5173/admin
- 管理员登录：`admin` / `admin123`

> 单端口替代方案：在 `client` 目录执行 `npm run build` 后，直接启动后端
> （`server` 目录 `npm start`），浏览器打开 **http://localhost:3000** 即是完整网站，
> 无需再单独启动前端、无需代理。详见 `docs/部署交付说明.md` 第 7 节。

## 常见问题

### Q: 应该从开发电脑拷贝哪些内容？
带走 `client/`、`server/`、`database/`、`docs/`、`README.md`、`.gitignore`。
不要带 `node_modules`、`client/dist`、`server/uploads`、`server/.env`。更完整的说明见 `docs/部署交付说明.md`。

### Q: npm install 很慢？
```bash
npm config set registry https://registry.npmmirror.com
```

### Q: MySQL 连接失败？
检查 `.env` 中的 `DB_PASSWORD` 是否正确，MySQL 服务是否已启动。

### Q: 端口被占用？
修改 `server/.env` 中的 `PORT`，或修改 `client/vite.config.js` 中的 `server.port`。

### Q: 前端页面空白？
1. 确保后端已启动，且 `vite.config.js` 中的代理配置正确；
2. 若只有某台机器/某个浏览器空白，多为本地登录缓存损坏所致——现已做容错（脏数据会被自动清除并正常渲染）；仍空白时可在浏览器开发者工具 Application → Local Storage 清除本站数据后刷新。
